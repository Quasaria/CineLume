À propos de CineLume

Publié le 1er octobre 2025 · Mis à jour le __BUILD_DATE_HUMAN_FR__

L'idée

CineLume est né d'un constat simple : il n'existait aucune application qui réponde directement à ma question quotidienne, qu'est-ce qui sort cette semaine au cinéma que j'ai envie de voir ?

Les applications dominantes sont organisées autour d'autres usages : Letterboxd est un journal social de visionnage, Trakt est un tracker épisodique avec compte payant, IMDb est une base de données saturée de publicités. Aucune n'optimise l'usage "agenda hebdomadaire des sorties + watchlist personnelle sans compte".

Trois besoins concrets

  1. Voir ce qui sort cette semaine au cinéma, filtré par pays
  2. Marquer les films qui m'intéressent (heart / bookmark) sans compte ni inscription
  3. Recevoir une notification le jour de leur sortie

Tout le reste de l'application — listes personnalisées, mode sélecteur swipe, pioche aléatoire, import Letterboxd, partage par lien — est dérivé organiquement de ces trois besoins, pas plaqué pour faire joli.

Choix techniques délibérés

Pas de backend

CineLume n'a pas de serveur. Toutes les données utilisateur (favoris, watchlist, listes, préférences) sont stockées dans le localStorage du navigateur. Aucune base de données. Aucun endpoint API privé. Cette contrainte rend impossible le tracking, la fuite de données, et les modèles commerciaux basés sur la pub ou les abonnements.

Pas de publicité

Aucune bannière. Aucun interstitiel. Aucun pixel publicitaire. Aucune télémétrie. L'expérience est entièrement épurée. Le coût d'infra étant nul (GitHub Pages héberge gratuitement), il n'y a pas de raison d'imposer de la pub.

Pas d'inscription

Aucun email, aucun mot de passe, aucun compte. Tu ouvres l'app et tu l'utilises immédiatement. Comme tout est local, tu peux exporter ta watchlist au format Letterboxd CSV à tout moment pour la migrer ailleurs, ou la partager par lien.

Progressive Web App

CineLume fonctionne dans n'importe quel navigateur moderne et peut être installée sur l'écran d'accueil iOS, Android, Windows, macOS et Linux. Mode hors-ligne via Service Worker (Workbox). L'expérience installée est équivalente à une app native.

Open source

Le code est public sous licence MIT, auditable et forkable. Aucune obfuscation, aucune télémétrie cachée. Si je disparais demain, n'importe qui peut continuer le projet ou faire le sien.

Sources de données

Les informations sur les films viennent de TMDB (The Movie Database), une base communautaire ouverte créée en 2008 qui couvre plus de 800 000 films. Les disponibilités sur les plateformes de streaming sont fournies par JustWatch via TMDB.

Aucune donnée n'est inventée ou scrappée : tout passe par les APIs publiques officielles.

Stack technique

React 19 · Vite 7 · TypeScript · Tailwind CSS · Zustand · TanStack Query · Framer Motion · i18next · Workbox PWA · HashRouter · Sonner.

Code splitting des modales (lazy loading + preload au idle), service worker NetworkFirst pour les API TMDB, CacheFirst pour les images. Bundle initial ~190 KB gzip.
